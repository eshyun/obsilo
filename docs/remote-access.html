Remote Access

Access your Obsidian vault from AI assistants like claude.ai, ChatGPT, and Cursor — from any device, anywhere. Obsilo deploys a relay server to your own Cloudflare account. Your data stays on your infrastructure.

How it works: Obsilo connects to a small relay server on your Cloudflare account. AI assistants send requests to the relay, which forwards them to your local Obsidian. No ports to open, no tunnels to install.

Setup (5 minutes)

Step 1: Create a Cloudflare account

Go to dash.cloudflare.com/sign-up and create a free account. You do not need the paid plan.

Step 2: Create an API token

  1. Go to API Tokens and click Create Token.
  2. Scroll to the bottom and click Create Custom Token.
  3. Add two permissions:
    • Account / Workers Scripts / Edit
    • Account / Account Settings / Read
  4. Under Account Resources, select All accounts.
  5. Remove Zone Resources (click the X next to it).
  6. Click Continue to summary, then Create Token.
  7. Copy the token (you will only see it once).

Step 3: Deploy the relay

  1. In Obsidian, go to Settings → Obsilo → Connections.
  2. Under Remote access, enable the toggle.
  3. Paste your Cloudflare API token.
  4. Click Deploy. This takes about 10 seconds.
  5. Once deployed, click Copy URL.

Step 4: Connect your AI assistant

Use the copied URL to set up the connector in your AI assistant. The URL includes an authentication token — do not share it publicly.

claude.ai Setup

  1. Go to claude.ai → Settings → Integrations.
  2. Click Add custom connector (or Add integration).
  3. Enter a name (e.g. "Obsilo Remote").
  4. Paste the connector URL from Obsilo.
  5. Set authentication to None (the token is embedded in the URL).
  6. Confirm the security warning and click Create.

The Obsilo tools are now available in every Claude conversation. Claude will automatically discover and use them.

ChatGPT Setup

  1. Go to chatgpt.com → Settings → Apps (or Plugins).
  2. Click Add custom app (or New App).
  3. Enter a name (e.g. "Obsilo Remote").
  4. Paste the connector URL from Obsilo.
  5. Set authentication to None.
  6. Confirm the warning and click Create.
ChatGPT requires manual activation. After creating the app, you must add it to each conversation by clicking the + button in the chat input and selecting "Obsilo Remote". Unlike claude.ai, ChatGPT does not auto-discover connectors.

Cursor / Windsurf Setup

  1. Open Cursor/Windsurf settings → MCP Servers.
  2. Click Add remote server.
  3. Paste the connector URL from Obsilo.

Available Tools

AI assistants connected via remote access have access to these tools:

ToolDescription
get_contextLoad user profile, memory, vault stats, skills, and rules. Should be called first.
search_vaultFull 4-stage search: semantic + keyword + graph expansion + reranking.
read_notesRead one or more notes by path, including frontmatter and metadata.
write_vaultCreate, edit, append, or delete files in the vault.
update_memoryAppend observations to Obsilo's memory files.
sync_sessionSave the conversation transcript to Obsilo's chat history.
execute_vault_opRun vault operations (canvas, frontmatter, etc.).

Updating the Relay

When Obsilo is updated with new relay features, click Redeploy in the remote access settings. This pushes the latest relay code to Cloudflare without changing the connector URL — no need to update your AI assistant configuration.

Troubleshooting

Tool execution fails ("Error occurred during tool execution")

  • Is Obsidian running? The relay forwards requests to your local Obsidian. If Obsidian is closed, tools will fail.
  • Is the remote access toggle enabled? Check Settings → Obsilo → Connections → Remote access.
  • Does the status show "Connected"? If not, click "Connect" or reload Obsidian.
  • Did you reset and redeploy? Reset generates a new token. You must update the URL in your AI assistant. Use "Redeploy" instead to keep the same URL.

401 Unauthorized errors in Obsidian console

The token stored in Obsilo does not match the token on Cloudflare. This happens after a Reset + Deploy if the old token was still cached. Solution: Reset the relay, Deploy again, then update the connector URL in your AI assistant.

Tools are discovered but never execute

The AI assistant can see tool definitions (served by Cloudflare) but execution requires the local Obsidian to be running and polling. Check the Obsidian developer console (Cmd+Option+I / Ctrl+Shift+I) for [RelayClient] messages.

ChatGPT ignores folder preferences

ChatGPT sometimes ignores the default folder specified in tool descriptions. This is a ChatGPT behavior limitation, not an Obsilo bug. You can work around it by explicitly stating the folder in your message (e.g. "save it in Inbox/").

First poll returns 404 after Obsidian restart

This is normal. The first poll after a restart may hit an old Cloudflare cache. The next poll succeeds. No action needed.

Security

  • Self-hosted: The relay runs on your own Cloudflare account. No data flows through third parties.
  • Token-based auth: Every request must include the correct token. The token is embedded in the URL.
  • No data storage: The relay is a pass-through. It does not store any vault data.
  • Encrypted in transit: All communication uses HTTPS/TLS (enforced by Cloudflare).
  • Free tier: Cloudflare Workers free tier includes Durable Objects. No payment required.
Keep your connector URL private. Anyone with the URL can access your vault tools. If you suspect the URL was compromised, click "Reset relay" in Obsilo settings and redeploy to generate a new token.
