Tools Reference

Complete reference for all tools available to Obsilo Agent. Tools are executable functions grouped by category.

Read Tools

These tools are read-only and are in the read tool group. Auto-approved by default.

read
read_file
Read the complete content of a file. Always call this before editing an existing file.
read
list_files
List files and folders in a directory. Use "/" for the vault root. Optional recursive parameter.
read
search_files
Full-text and regex search across files. Returns matching lines with file path and line number. Supports file_pattern glob filter.
read
read_document
Parses and extracts text content from office documents (PPTX, XLSX, DOCX), PDFs, and data formats (JSON, XML, CSV). Returns structured text with metadata. Supports all common document formats in the vault.

Parameters: read_file

ParameterTypeRequiredDescription
pathstringyesVault-relative path to the file (e.g., "Projects/plan.md")

Parameters: list_files

ParameterTypeRequiredDescription
pathstringyesDirectory path. Use "/" for vault root
recursivebooleannoInclude subdirectory contents (default: false)

Parameters: search_files

ParameterTypeRequiredDescription
pathstringyesDirectory to search in
patternstringyesText or regex pattern to search for
file_patternstringnoGlob pattern to filter files (e.g., "*.md")

Vault Intelligence Tools

These tools are read-only metadata and search operations. In the vault tool group. Auto-approved by default.

vault
get_vault_stats
Returns a vault overview: total notes, folder tree, top tags, recently modified files. No parameters needed.
vault
get_frontmatter
Read all YAML frontmatter fields of a note (tags, aliases, dates, custom properties).
vault
search_by_tag
Find all notes with given tags. Supports AND or OR matching. Tags with or without # both work.
vault
get_linked_notes
Get forward links and backlinks for a note. Direction: "both" (default), "forward", or "backlinks".
vault
get_daily_note
Read a daily note. offset=0 (today), -1 (yesterday), 1 (tomorrow). create=true creates it if missing.
vault
open_note
Open a note in the Obsidian editor view. Use after creating or editing a note to bring it into focus.
vault
semantic_search
Hybrid semantic + keyword search. Finds notes by meaning. Requires Semantic Index to be built. Supports folder, tag, and date filters.
vault
query_base
Query an Obsidian Bases (.base) file. Returns the notes matching the view's filter conditions.

Parameters: semantic_search

ParameterTypeRequiredDescription
querystringyesNatural language search query
top_knumbernoMax results to return (default: 8, max: 20)
folderstringnoRestrict results to notes in this folder (prefix match)
tagsstring[]noOnly return notes that have ANY of these tags
sincestringnoOnly return notes modified on or after this date (ISO format: "2025-01-01")

Writing & Editing Tools

These tools modify vault files and require approval (or auto-approval). In the edit tool group.

edit

Create a new file or completely replace an existing file's content. Preferred for new notes or full rewrites.
edit

Replace a specific string in an existing file. Safer than write_file for targeted changes — preserves surrounding content.
edit
append_to_file
Append content to the end of a file. Ideal for daily notes, logs, and additive entries. Optional separator before appended content.
edit
update_frontmatter
Set or update YAML frontmatter fields. Preserves all existing fields. Creates frontmatter if none exists.
edit
create_folder
Create a folder, including all parent folders in the path.
edit
delete_file
Move a file or empty folder to the Obsidian trash (recoverable). Does not permanently delete.
edit
move_file
Move or rename a file or folder. Updates wikilinks automatically via Obsidian's link resolver.
edit
create_excalidraw
Create an Excalidraw drawing (.excalidraw.md) with specified elements. Requires the Excalidraw plugin to be installed.
edit
create_docx
Creates Word documents (.docx) with structured sections, headings, paragraphs, bullet lists, and styling options (fonts, colors, spacing). Up to 100 sections per document.
edit
create_pptx
Creates PowerPoint presentations (.pptx). Template mode or Adhoc mode. Up to 50 slides.
edit
create_xlsx

edit

edit
plan_presentation
Plans a presentation from source material using an ingested corporate template.

Office Document Creation

These tools create binary office documents directly in the vault. In the edit tool group. Require approval.

edit
create_docx
Creates Word documents (.docx) with structured sections, headings, paragraphs, bullet lists, and styling options (fonts, colors, spacing). Up to 100 sections per document.
edit
create_pptx
Creates PowerPoint presentations (.pptx). Two modes: Template mode (uses slides from an ingested corporate template with shape-level content from plan_presentation) and Adhoc mode (direct slide content with title, body, notes, and theme). Up to 50 slides per presentation.
edit
create_xlsx
Creates Excel spreadsheets (.xlsx) with multiple sheets, headers, data rows, column widths, and basic formatting. Up to 20 sheets per workbook.
edit
ingest_template
Analyzes a corporate PowerPoint template (.pptx) and generates a slide-type catalog. Extracts shape names, types, and positions per slide, then groups slides by their PowerPoint layout name into reusable slide types. Run once per corporate template.
edit
plan_presentation
Plans a complete presentation from source material using a corporate template. Uses an internal LLM call to generate slide-by-slide content matched to template shapes. Returns a deck plan ready for create_pptx. Always call this before create_pptx when using corporate templates.

ParameterTypeRequiredDescription

slidesarrayyesArray of slide objects with title, body, and speaker notes. Up to 50 slides.
themeobjectnoTheme colors and layout options

Parameters: create_xlsx

ParameterTypeRequiredDescription

sheetsarrayyesArray of sheet objects with name, headers, rows, and column widths. Up to 20 sheets.

Web Tools

Require Web Tools to be enabled in Settings → Web Search. In the web tool group.

web
web_fetch
Fetch a URL and return its content as Markdown. Handles pagination via startIndex. Max 20,000 chars by default.
web
web_search
Search the web and return titles, URLs, and snippets. Requires Brave or Tavily API key. Follow up with web_fetch to read full pages.

Agent Control Tools

These tools control the agent loop. In the agent tool group.

agent
ask_followup_question
Ask the user a clarifying question with optional answer choices. Pauses the loop until the user responds.
agent
attempt_completion
Signal that the task is complete. Call this AFTER streaming your full response as text. The result field is an internal log entry, not shown to the user.
agent
update_todo_list
Publish a task plan as a visible checklist. Use only for tasks with 3+ distinct steps. Statuses: pending ([ ]), in-progress ([~]), done ([x]).
agent
switch_mode
Switch the active mode mid-conversation. Requires user approval by default.
agent
new_task
Spawn a sub-agent with its own context window. The child runs in a completely fresh conversation history — its API calls consume only its own token budget, not the parent's. Available in Agent mode only. Pass all necessary context in the message — the sub-agent cannot see the parent conversation.
agent
update_settings
Modify plugin settings or apply a preset (permissive, balanced, restrictive). Used during onboarding and for runtime configuration changes.
agent
configure_model
Add, select, or test an AI model configuration. Used during onboarding to set up the user's first model connection.
agent
evaluate_expression
Execute TypeScript in an isolated sandbox. Best for: text/JSON processing, vault batch operations, computations, HTTP API calls. Not suited for binary file generation (DOCX, PPTX, XLSX) which requires Node.js APIs unavailable in the sandbox.
agent
manage_skill
Create, update, or delete skills. Skills are persistent instruction sets (Markdown) that guide the agent for specific task types. They are not executable code.
agent
manage_mcp_server
Connect, disconnect, or reconfigure MCP servers at runtime. Allows the agent to dynamically extend its tool set by managing external tool servers.
agent
manage_source
Manage source code modules for self-development. Create, update, or delete code files used by dynamic tools and agent extensions.
agent
read_agent_logs
Read the agent's operation audit logs. Useful for debugging, reviewing past tool executions, and understanding agent behavior.

Sub-Agent Workflows (new_task)

The new_task tool enables multi-agent workflows by spawning child agents that operate outside the parent's token context window. Each sub-agent gets a completely fresh conversation history, runs its task independently, and only returns a compact text result to the parent. This keeps the main agent's context clean for high-level reasoning while offloading detailed work.

How it works

  1. The parent agent calls new_task with a mode and a task description.
  2. A new AgentTask is created with an empty conversation history — no shared token budget with the parent.
  3. The child agent runs its own API loop with full tool access (or read-only, depending on mode).
  4. When done, only the child's accumulated text result flows back into the parent's context as a single tool result.
  5. Depth-guarded: Nesting is limited — children at maximum depth cannot spawn further sub-agents, preventing runaway recursion.

Parameters

ParameterTypeRequiredDescription
modestringyes"agent" (full capabilities: read, write, web) or "ask" (read-only vault queries and search)
messagestringyesFull task description with all necessary context. The sub-agent cannot see the parent conversation — include everything it needs.

Supported patterns

Orchestrator-Worker
Parent delegates specialized subtasks to focused sub-agents. Each worker handles one concern (e.g., research, writing, analysis) and reports back.
Prompt Chaining
Sequential sub-agents where each builds on the previous result. Useful for multi-stage pipelines (e.g., gather data, then analyze, then summarize).
Evaluator-Optimizer
One sub-agent generates content, another reviews and critiques it. The parent iterates until quality criteria are met.
Routing
Parent dispatches to the right sub-agent based on task type. Different modes or instructions per sub-agent allow specialization.

When to use sub-agents

  • Task requires 5+ steps across different specialties
  • Context isolation helps — e.g., deep research into many files that would bloat the main context
  • Truly parallel independent subtasks

For simple read/write operations, file conversions, or single-tool calls, the main agent should use its own tools directly — no need for a sub-agent.

Plugin Integration Tools

These tools interact with installed Obsidian plugins via VaultDNA. In the plugin integration tool group.

skill
execute_command
Execute an Obsidian command by its command ID. Covers core commands and commands from installed community plugins.
skill
call_plugin_api
Call a method on an installed plugin's API. Protected by an allowlist — only safe methods can be invoked without override.
skill
execute_recipe
Run a predefined multi-step recipe (e.g., Pandoc export, Dataview query). Recipes combine multiple plugin commands into a single action.
skill
resolve_capability_gap
When the agent needs a capability not available, this tool suggests and can install a community plugin to fill the gap.
skill
enable_plugin
Enable or disable an installed Obsidian plugin. Used by the agent to activate plugins it needs for a task.
skill
render_presentation
Renders a PPTX presentation to images using LibreOffice. Returns slide images so the agent can visually inspect the result for text overflow, layout problems, or design issues. Requires LibreOffice. Use after creating a presentation with create_pptx.
